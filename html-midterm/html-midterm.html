  1. Line 1: On the first line, I added <!DOCTYPE html> so the browser knows what document type to expect.
  2. Line 2: <html lang="en"> is in the wrong spot. I deleted it from the header area and replaced the first <html> with <html lang="en">.

  4. Line 7: At the end of the head, it needs a closing tag. I added </head> to fix this.
  5. Line 10: The "Welcome to My Error-Filled Page" heading needs a closing tag at the end. I added </h1> to the end of the line.
  6. Line 12: The "This is a paragraph with mismatched tags and missing closing tags." text needs an opening tag. I added <p> at the beginning of the paragraph to fix this.
  7. Line 14-25: The tags opening and closing the list do not match. It must either be an ordered list with <ol> or an unordered list with <ul>. I changed the tag on line 14 from <ol> to <ul> to match the </ul> on line 25 to fix this.
  8. line 27: The "This is another paragraph without proper indentation." text needs a closing tag. I added </p> to the end to fix this.
  9. Line 29: The <img> tag needs alternative text. I added alt="The two states of every programmer meme image" right before the > to fix this. This is necessary to help the visually impaired, to optimize search engines, and to show the text if the image fails to load.

  11. Line 40: The wrong syntax was used for the table data cell element containing "25". It needs a / to show that it is a closing tag. I replaced <td> with </td> to fix this.

  15. Line 52: The bgcolor attribute is not supported in HTML5. To fix this, I replaced bgcolor="lightgray" with style="background-color: lightgray;" to properly set the text background to lightgray.
  16. Line 56: The comment has not been properly closed, so the rest of the html after line 54 had been commented out. To close the comment, I added --> to the end of the line.
